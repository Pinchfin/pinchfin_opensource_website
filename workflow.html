Execution Framework

Sequential execution with enforced dependency gates.

The pinchfin workflow orchestrates 229+ skills across 17 sequential phases, organized into 4 execution segments. Each phase produces structured intermediate output consumed by downstream phases. Four blocking gates enforce data quality before the pipeline advances. No phase may execute until its declared dependencies are satisfied.

Segments enable multi-session execution: Foundation (Phases 1-4), Analysis (Phases 5-9), Synthesis (Phases 9-11), and Decision (Phases 12-15). State files persist between sessions. Each segment produces a validated output document.
