MCP Integration Guide

Getting Started with pinchfin MCP

Overview

pinchfin is delivered as an MCP (Model Context Protocol) server that integrates directly into Claude-native analytical environments. The MCP provides access to the complete pinchfin credit operating system — 24 modules, 229+ analytical skills, 81+ governance nodes, and the full 17-phase underwriting pipeline — without requiring local installation or system configuration.

What You Need

  • A Claude Pro, Team, or Enterprise account with MCP support enabled
  • An active pinchfin access subscription
  • Your pinchfin API credentials (provided upon registration)

User Responsibilities

pinchfin is a tool, not a substitute for professional judgment. Users are responsible for:

  • Data Accuracy — All source documents (rent rolls, T-12 statements, appraisals) must be verified before upload. The system processes what you provide; garbage in produces garbage out.
  • Confidentiality — Deal data, sponsor information, and financial records are sensitive. Users must handle all inputs and outputs in accordance with their firm's confidentiality policies and any applicable NDA obligations.
  • API Key Management — Protect your pinchfin API credentials and any integrated data service keys. Do not share credentials or embed them in shared repositories.
  • Output Verification — All analytical outputs require independent verification by qualified credit professionals. The platform assists analysis; it does not replace the human credit decision.
  • Regulatory Compliance — Users must ensure that their use of the platform complies with all applicable lending regulations, fair lending requirements, and supervisory guidance in their jurisdiction.

Quick Start

Step 1: Register for Access

Contact bmurphy@veletacapital.com to request access. Upon registration, you will receive:

  • Your pinchfin MCP server endpoint
  • API authentication credentials
  • Your default credit configuration profile

Step 2: Connect the MCP Server

Add the pinchfin MCP server to your Claude environment:

Claude Desktop (claude_desktop_config.json):

{
  "mcpServers": {
    "pinchfin": {
      "url": "https://mcp.pinchfin.com/sse",
      "headers": {
        "Authorization": "Bearer YOUR_API_KEY"
      }
    }
  }
}

Claude Code:

claude mcp add pinchfin --transport sse https://mcp.pinchfin.com/sse

Step 3: Verify Connection

Once connected, verify the integration by asking Claude:

“What pinchfin tools are available?”

Claude should return the full catalog of available analytical skills and modules.

Step 4: Run Your First Analysis

Upload a rent roll and trailing twelve-month operating statement, then instruct Claude:

“Run a pinchfin underwriting analysis on these documents.”

The platform will initiate the 17-phase pipeline, processing your transaction through data infrastructure, sponsor due diligence, cash flow underwriting, constraint waterfall sizing, and deliverable generation.

Uploading Comparable Sales

Comparable sale and lease data can be uploaded to enhance the underwriting rent determination and valuation analysis.

Upload Format

Provide comparable data as structured CSV or JSON with the following fields:

  • Required: Property address, sale or lease date, price or rent per unit/SF, unit count, property type, data source
  • Optional: Cap rate, occupancy at sale, buyer/seller identity, year built, renovation status

How Comps Are Processed

Uploaded comps are geocoded, validated against assessor records where available, and normalized to per-unit and per-SF metrics. The system applies distance-weighted matching (proximity to subject), vintage adjustments (time since transaction), and property class alignment scoring. Duplicates are flagged automatically.

Integration with Analysis

Validated comps flow into the comparable analysis report and the lender rent determination module (Phase 6A). All comp selections are documented with rationale, and vintage adjustments are applied based on submarket trend data from the proprietary data services layer.
