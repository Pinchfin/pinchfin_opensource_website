System Architecture

An AI-native operating system for real estate credit.

pinchfin architecturally separates deterministic financial computation from artificial inference, enforces execution discipline through a multi-phase orchestration framework governed by enterprise nodes, and codifies every constraint, threshold, and underwriting assumption in explicit, version-controlled configuration. 229+ sequentially executed skills produce structured credit analysis across 24+ autonomous modules.

Deterministic Engine

Debt service coverage, constraint waterfall sizing, cash flow projections, and amortization schedules execute in deterministic Python. No generative model touches the computation layer. Identical inputs produce identical outputs. Fully auditable and independently testable.

Constraint-Based Sizing

A five-constraint waterfall sizes every loan: loan-to-value (as-is and stabilized), debt service coverage ratio, debt yield, loan-to-cost, and maximum proceeds cap. The binding constraint governs maximum loan amount. No pre-assumption about which constraint will bind — the full waterfall executes on every transaction.

Credit Box Config

All credit parameters, policy thresholds, and constraint hierarchies reside in a centralized, version-controlled configuration file. A single parameter change propagates through every downstream sizing, coverage, and yield calculation. Audit-trailed and committee-reviewable.

Natural Language Control

Credit committee and senior management modify lending policy through natural language directives. The system translates those directives into configuration updates, recalculates all affected analyses, and propagates changes to downstream workflows — without code modification or redeployment.
